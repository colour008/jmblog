



# Cloudflare部署GitHub静态页面部署命令

## 一、部署已打包（或构建）好的项目

**方法1.** 在部署命令内直接添加如下内容并删除路径的`/`即可部署成功。

```bash
# assets就是网站项目目录（./表示根目录），name后是你的项目名称，后面的日期写成当日即可
npx wrangler deploy --assets=./ --name=colour008_blog --compatibility-date=2025-12-14
```

![image-20260103231651082](https://bu.dusays.com/2026/01/03/695932e417e1a.png)

**方法2.** 在项目中创建 `wrangler.jsonc` 配置文件，文件内容如下

```jsonc
{
  "name": "colour008_blog",
  "compatibility_date": "2025-12-14",
  "assets": {
    "directory": "./"
  }
}
```
配置完成后在部署中可执行简化的命令：

   ```bash
   npx wrangler deploy
   ```

## 二、配置vitepress类项目

在构建命令输入`npm run docs:build`，然后在部署命令输入如下内容即可部署成功

```bash
# assets就是网站项目目录（.vitepress/dist表示打包后的目录），name后是你的项目名称，后面的日期写成当日即可
npx wrangler deploy --assets=.vitepress/dist --name=docs --compatibility-date=2026-01-03
```

![image-20260103232105821](https://bu.dusays.com/2026/01/03/695933e2d0db8.png)

同样也可以在项目中创建`wrangler.jsonc` 配置文件，具体参照上述方法2即可。
